### 1. Full Stack, bukan Frontend {#1-fullstack-instead-of-frontend}

Sebagian besar dari Next.js merupakan kerangka kerja frontend. Dirancang untuk membangun sebuah
frontend yang terhubung ke beberapa API lainnya. Next.js tidak dirancang secara default
untuk membangun sebuah aplikasi fullstack yang sesungguhnya.

Blitz menambahkan semua fitur dan fungsionalitas yang hilang yang mengubah Next menjadi 
kerangka kerja fullstack yang sebenarnya. Fitur utama ini mencakup akses database 
secara langsung, middleware, dan otentikasi.

### 2. Lapisan Data {#2-data-layer}

Lapisan data baru Blitz adalah sebuah teknologi baru untuk menghilangkan API 
sambil mempertahankan client-side rendering. Dalam sebuah aplikasi Blitz, kamu dapat mengimpor
kode servermu langsung menuju frontend sehingga kamu tidak perlu membuat sebuah API
dan melakukan pengambilan data dari frontend. Pada saat build time, Blitz akan secara otomatis
memasukan sebuah API call yang menjalankan kode kode server di server. Pada dasarnya,
Blitz mengabstraksi API-mu menjadi sebuah langkah compile.

Ini adalah sebuah game changer untuk pengembangan aplikasi React karena menghilangkan
seluruh bagian dari arsitektur tradisional aplikasi React. ini berarti
lebih mudah dipelajari, lebih cepat dikembangkan, dan lebih menyenangkan untuk membuat sesuatu!

Meskipun demikian, kamu tetap dapat terus melakukan pengambilan data melalui REST atau GraphQL 
seperti yang selalu kamu lakukan. Blitz tidak membatasi ini dengan cara apa pun.

### 3. Otentikasi Bawaan {#3-built-in-authentication}

Blitz memiliki session management bawaan yang berfungsi dengan identity provider 
apa pun, termasuk email/kata sandi yang dihosting sendiri dan layanan pihak ketiga.
Otentikasi adalah sesuatu yang rumit dan sulit diterapkan dengan benar. 
Memilikinya secara bawaan dalam Blitz akan menghemat banyak waktu dan merendahkan 
potensi kerentanan keamanan.

Membangun UX yang mengagumkan dengan auth di Next.js sangat rumit dan membosankan, 
tetapi Blitz memberikan-mu DX terbaik di kelasnya secara gratis.

### 4. Konvensi {#4-conventions}

Next.js tidak memiliki konvensi selain dari perutean berbasis sistem file. 
Hal ini membuat setiap pengembang menciptakan kembali roda untuk setiap proyek.

Blitz menghadirkan sejumlah konvensi untuk hal-hal seperti cara mengatur file-mu dan 
cara menyusun rutenya. Ini berarti kamu tperlu idak membuang waktu memikirkan hal ini untuk 
setiap proyek. Ini juga sangat menguntungkan komunitas. Struktur proyek dan pola arsitektur 
yang umum memungkinkan-mu untuk berpindah dari aplikasi sebuah Blitz ke aplikasi Blitz l
ainnya dengan nyaman dan mudah.

### 5. Perancah Kode {#5-code-scaffolding}

Ada dua cara utama untuk mengurangi jumlah kode yang harus kamu tulis 
dengan tangan:

1. Code generation
2. Code scaffolding

**Code generation** berarti sebuah library men-generate sebuah kode untukmu. 
Contohnya adalah graphql-code-generator yang menghasilkan kode dari kueri GraphQL 
dan Hasura yang menghasilkan seluruh API GraphQL dari skema database milikmu.
Biasanya codegen tidak memiliki cara untuk menyesuaikan kode yang dihasilkan sepenuhnya. 
Kamu hanya bisa bergantung pada library tersebut dengan apa yang didukungnya. Seringkali juga 
kamu akan mengalami kebuntuan karena codegen tidak memiliki solusi untuk itu. 
Dan kamu tidak dapat memperbaikinya karena kamutidak memiliki kodenya.

**Code scaffolding** berarti kode awal di-scaffold ke dalam proyekmu dan untukmuy. 
Sejak saat itu, kamu memiliki kepemilikan penuh atas semua kode dan dapat 
menyesuaikannya sebanyak yang kamu butuhkan. Kelemahan dari perancah kode adalah 
kamu tidak mendapatkan pembaruan otomatis seperti yang kamu lakukan dengan pembuatan 
kode dari library pihak ketiga. Tetapi keuntungan terbesarnya adalah kamu dijamin 
tidak akan pernah dibatasi oleh pilihan desain orang lain yang tidak dapat kamu ubah.

Blitz adalah penggemar berat dari code scaffolding. Kami memiliki banyak sekali perintah
[`blitz generate`](./cli-generate) untuk scaffolding code kedalam
proyekmu. Memang masih terlalu awal untuk code scaffolding milik kami â€” dan kami memiliki 
banyak fitur canggih yang masih perlu ditambahkan, termasuk kemampuan untuk menambahkan 
templat perancah kustom-mu sendiri.

### 6. Resep {#6-recipes}

Resep adalah perintah satu baris untuk men-scaffolding sebuah kode ke dalam proyekmu
dari resep MDX di npm atau repositori git.

Contoh:

- `blitz install tailwind` - satu perintah untuk menginstal dan mengkonfigurasi tailwind
- `blitz install chakra-ui` - satu perintah untuk menginstal dan mengkonfigurasi chakra
- `blitz install material-ui` - satu perintah untuk menginstal dan mengkonfigurasi material-ui

Resep sangatlah kuat. Mereka dapat mengubah hampir semua hal dalam proyekmu, termasuk 
menambahkan dependensi, mengubah kode, menambahkan kode, dll. Mereka ditulis melalui 
MDX dan dapat disusun seperti komponen React.

### 7. Pengembangan Aplikasi Baru {#7-new-app-development}

Aplikasi Next.js baru benar-benar sederhana. Jadi setiap kali kamu memulai proyek 
Next.js, kamu harus menghabiskan berjam-jam menyiapkan semua dasar seperti eslint, 
prettier, husky git hooks, dll.

Aplikasi Blitz baru menghemat banyak waktu karena semua itu sudah dikonfigurasi 
sebelumnya untukmu! Yang pasti kamu selalu dapat menyesuaikannya nanti, tetapi 
memiliki titik awal yang berfungsi dan siap digunakan itu luar biasa loh.

### 8. Batasan Santai {#8-relaxed-restrictions}

Next.js mengharuskan semua halaman React-mu masuk dalam satu folder `pages/`, 
namun Blitz melonggarkan batasan ini dengan memungkinkan kamu untuk memiliki 
folder `pages/` yang tidak terbatas.

Untuk proyek besar, ini merupakan keuntungan besar karena kamu dapat memisahkan 
dan mengatur halaman sesuai keinginanmu.

### 9. Route Manifest {#9-route-manifest}

Next.js requires you to manually type out page locations. Blitz comes with
a [Route Manifest](./route-manifest), so you can do:

```ts
<Link href={Routes.ProductsPage({ productId: 123 })} />
// instead of
<Link href={`/products/${123}`} />
```

This improves expressiveness and simplifies moving pages to other
location.
